# Methodology

## Problem: Die-Link Chronology Reconstruction

Ancient coins were struck from paired dies — an obverse (front) and reverse (back). As dies wore out during production, mint workers replaced them sequentially. Numismatists reconstruct this production timeline through **die-link analysis**:

1. **Coin pairing evidence**: If coin A uses obverse die O1 + reverse R3, and coin B uses O1 + R7, then O1 was active when both R3 and R7 were in use.
2. **Wear transfer evidence**: When a worn die strikes a fresh die, characteristic wear patterns transfer. If wear on die O5 matches wear first seen on O4's coins, then O4 preceded O5.

The result is a **die-link network** — a graph where nodes are dies and edges encode co-occurrence and temporal precedence.

## Graph Representation

### Nodes (Dies)

Each die \( d_i \) has features:

- `wear_index` ∈ [0, 1] — proxy from coin weights and die degradation
- `side` — obverse (1) or reverse (0)
- `style_vector` ∈ ℝ⁸ — averaged stylistic attributes from linked coins

### Edges

- **Coin-link edges** (undirected): connect obverse-reverse pairs observed on the same coin
- **Wear-transfer edges** (directed): \( d_i \rightarrow d_j \) when wear evidence suggests \( d_i \) was used before \( d_j \)

## Classical Baselines

### Wear Index Ordering

Sort dies by ascending wear index. Naive but surprisingly effective for obverse dies in sequential production.

### Topological Sort

Build a DAG from wear-transfer edges and apply topological sorting. Handles explicit precedence constraints but fails on cycles (common with reverse die reuse).

## Mint Atlas GNN

### Architecture

```
Input features (10-dim) → GATConv (4 heads, 64-dim) → BatchNorm → ELU
                        → GraphSAGE (32-dim embeddings)
                        → Pairwise MLP classifier (temporal ordering)
                        → Sequence ranker (auxiliary)
```

### Training Objectives

**Primary — Pairwise temporal loss:**

For each die pair \((d_a, d_b)\) with known ordering, predict whether \( d_a \prec d_b \):

\[
\mathcal{L}_{\text{pair}} = \text{BCE}(\text{MLP}([h_a; h_b; \Delta_{\text{wear}}]), y_{ab})
\]

**Auxiliary — Sequence ranking loss:**

\[
\mathcal{L}_{\text{rank}} = \text{CE}(\text{Ranker}(h_i), \text{seq}_i)
\]

**Combined:**

\[
\mathcal{L} = \mathcal{L}_{\text{pair}} + 0.3 \cdot \mathcal{L}_{\text{rank}}
\]

### Inference

At inference time, pairwise predictions are aggregated via tournament ranking: each die receives a score based on how many other dies it is predicted to precede.

## Evaluation

| Metric | Definition |
|--------|-----------|
| Kendall τ | Rank correlation between predicted and ground-truth orderings |
| Pairwise accuracy | Fraction of die pairs \((d_i, d_j)\) with correct relative order |

## Synthetic Data Generation

Training data is generated by simulating mint production:

1. Obverse dies are introduced sequentially (O000 → O001 → ...)
2. Reverse dies are reused across multiple obverse dies
3. Coin weights decrease with obverse die wear
4. Wear-transfer edges are inferred from the ground-truth sequence

This mimics the structure of real ancient mint production (e.g., Athenian owl tetradrachms, Seleucid coinage).
